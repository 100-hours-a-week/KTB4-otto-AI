## v6.1 — 318M instruction 튜닝 실측 (2026-06-25)
318M 베이스로 멀티태스크 SFT(`otto_gpt_350m_instruct.pt`, 4k step).
- **실측**: 지식→rag 3/3·날씨·검색 라우팅 정확(검색 "환율이" 안 붙음 = 57M보다 약간↑).
  그러나 **숫자 복사 여전히 실패**(`345→346`, `100→10`), 과트리거 잔존(`건강`→rag 인자 "양반").
- **핵심 교훈(정직)**: 라우팅은 작은 SFT 데이터의 단순 패턴 → **318M ≈ 57M**(이 task엔 베이스 향상 효과 미미).
  숫자 복사는 토크나이징 문제(용량 무관). 318M의 ppl 4.4 향상은 **지식·유창성**에서 나타남(이 라우팅 테스트론 안 보임).
- 실용: 에이전트 안전망(extract 인자 재추출 + safe_router)이 위 실패를 모두 흡수 → 318M_instruct 꽂으면 동작.
  318M 베이스의 진짜 활용처 = observation→answer 자연어 합성(추후) / 자유 생성.

## v6 — 모델 스케일업 350M (천장 올리기) (2026-06-24)
"시간 더 = 똑똑"이 아니라(수렴 후 0), **파라미터 = 지식 용량**이 천장이라는 결론 → 모델을 키움.
- **57M → 318M**(n_layer 10→24, n_embd 624→1024, n_head 12→16). GPT-2 medium 급, 5.6배.
- 별도 체크포인트 `otto_gpt_350m.pt`(+_best) — **기존 57M `otto_gpt.pt` 보존**. 처음부터 새 학습(fresh).
- 메모리 맞춘 배치(A100 b8/ga6, L4 b2/ga24, T4 b1/ga48 — A100 권장). `total_iters`=60k, max_iters=4k.
- 데이터 증량: 비게이트 소스당 200k→500k 문서(350M 먹이기).
- ⚠ 정직한 한계: 350M Chinchilla 적정 ≈70억 토큰인데 Colab 현실선 ~15억 → **다소 undertrained**.
  A100(Pro+)+장시간 필요. 그래도 57M보다 유창성·일반화 향상 기대. 초기엔 loss 높음(정상).
- **실측(Colab A100, 2026-06-24~25)**: 318.4M, 학습 곡선 4k→8k→20k→60k 에서 val_loss
  2.21→1.92→1.69→**1.48(ppl ≈4.4)**. 57M(ppl ~31) 대비 7배↓. 60k(목표 지평) 완주, 하락 완만 =
  좋은 정지점. 단 데이터도 변경(maywell)이라 순수 모델효과로 단정 불가. 다음: 생성품질 확인 →
  318M 베이스로 instruction 튜닝(라우터/챗봇).

## v5 — 데이터 확장 (이어서 사전학습) (2026-06-24)
위키(백과사전체)만으로 좁던 베이스를 다양한 한국어 텍스트로 넓혀 똑똑하게.
- 1차 시도 CulturaX(`uonlp/CulturaX`)는 **게이트(약관+HF토큰)** 라 사용자 환경에서 반복 실패 →
  **비게이트로 전환**: `maywell/korean_textbooks`(오픈, claude_evol+ko_wikidata) + `beomi/KoAlpaca-v1.1a`.
  (CulturaX 셀은 `USE_CULTURAX=False` 로 남겨둠 — 토큰 되면 재사용 가능)
- **자동화 수정**(수동 단계 제거): `total_iters` 40k→80k(이어학습 자동 실행), bin은 코퍼스
  변경 시 **자동 재생성**(수동 삭제 불필요 → 새 데이터 자동 반영). 토크나이저는 재사용.
- 실측 교훈: total_iters 미상향 시 0 step(헛돔) 확인 후 자동화. 위키 전용은 40k 에서 수렴
  (val_loss 3.42 정체) → **새 데이터 없이는 추가 학습 무의미**.
- ⚠ 실제 추가 학습은 Colab 진행 중. 57M 지식 천장은 남으나 유창성·라우팅 일반화 개선 기대.

## v4 — 데이터 개선 + RAG 도구 (2026-06-24)
v3 실측에서 드러난 약점(인자 오류·과트리거)을 데이터로 공략 + 지식 외주 경로 추가.
- **인자 정확도**: calculator 예시를 고정 4종 → **랜덤 숫자 400개(전부 고유)**. 패턴 암기가 아니라
  "숫자를 그대로 복사"하도록 학습 (v3 의 12*7→12*8 오류 대응).
- **과트리거 완화**: 도구 예시 반복 ×20 → **×3**, 도시/질의 다양화. 도구 비율 ~9%로 균형.
- **rag 도구 추가(★)**: 지식/사실 질문("X가 뭐야?")을 직접 답(헛소리)하지 말고 **`rag(질문="X")`**
  로 외부 DB(03_rag) 조회하게 학습. 20개 주제×5패턴.
- 데모 셀에 계산 정확도·지식→rag 라우팅·혼합 테스트 추가.
- 04_agent: 기본 레지스트리에 `rag`→`rag_search`(03_rag FastAPI HTTP) 추가.
- 검증(로컬): 데이터 생성(calc 400 고유, 지식→rag 포맷), 마스킹 SFT 동작, rag 라우팅
  (스텁 서버 답변 + 서버없음 graceful).
- **실측 결과(Colab, 2026-06-24)**:
  - 🟢 **지식→rag 라우팅 3/3 완벽** (인공지능/광합성/상대성이론 → rag). 헛소리 대신 외부 DB 조회.
  - 🟢 `12*7` 정확 (v3 오류 해결). 날씨/검색 라우팅 정상.
  - 🔴 **인자 정확도 여전히 약함**: `345+678`→`346+679`(3자리 +1), `오늘 환율`→`오늘 환율이`,
    `건강하게 사는 방법`→`rag(질문="인공학적")`(과트리거+인자 쓰레기).
  - 결론 강화: **라우팅은 우수, 정확한 인자 복사가 57M 한계.** → 인자는 에이전트 레이어에서
    원문 재추출로 보완(특히 숫자/calculator). 멀티스텝·더 긴 학습은 이후.
- **04_agent end-to-end 실측(진짜 otto, Colab, 2026-06-24)**: calculator/weather 라우팅 완벽.
  그러나 **"어댑터즈 구독료 얼마야?" → rag 라우팅 실패 + 환각**(학습 분포 밖: 미지의 고유명사 +
  "얼마야" 형태). 같은 질문을 규칙 라우터(local_router)는 rag 로 보내 정답 → **우리 모델이 규칙보다
  못한 케이스**. 57M 의 "모르면서 자신있게 직접 답" 성향이 핵심 약점.
  → v5 과제: rag 학습 예시를 다양한 사실질문 형태(얼마/가격/언제/어디 + 미지 고유명사)로 확대,
    "확신 없으면 rag" 바이어스. 단, 57M 의 환각 성향은 구조적 한계로 완전 해결은 어려움.

## v3 — 멀티태스크 instruction 튜닝 (2026-06-24)
- `finetune_instruct_colab.ipynb` 를 단일 태스크(KoAlpaca) → **멀티태스크**로 재구성.
- **통일 포맷** `### 지시 / ### 입력 / ### 응답` + **프롬프트 마스킹**: 응답 토큰만 loss
  (기존엔 프롬프트까지 학습했음 → ignore_index=-1 로 마스킹).
- 태스크 혼합: KoAlpaca(일반 대화/QA) + **도구 호출(tool-call) 합성**(weather/search/calculator,
  `tool(arg="...")` 형식) → "기능마다 다른 동작" + `04_agent` 로 가는 다리.
- 베이스를 `otto_gpt_best.pt`(추론용 best)에서 로드(없으면 구버전 호환).
- 검증(로컬 CPU): 포맷+마스킹(응답 토큰만 학습 확인), 도구합성, 실제 모델로 마스킹 SFT 1~3스텝 loss 감소.
- **실측 결과(Colab, 2026-06-24)**:
  - 🟢 도구 호출 3/3 완벽 — weather/search/calculator 형식·인자 정확. 에이전트 엔진 작동.
  - 🔴 일반 대화: 문법은 유창하나 **지식 헛소리**(57M 구조적 한계) + **도구 과트리거**(일반 질문을
    search 로 오라우팅). 후자는 도구 예시 ×20 과다 노출 탓 → v4 에서 ×3~5 로 재균형 + 직접답변
    네거티브 예시 추가 예정.
  - 전략적 결론: otto_gpt = "지식 모델"이 아니라 **"도구 라우터/컨트롤러"**. 지식은 RAG/큰 모델에 외주.
- **04_agent end-to-end 실측(Colab, 2026-06-24)**: 실제 `otto_gpt_instruct.pt` + 04_agent 루프.
  라우팅은 대체로 정확하나 **인자 복사 오류 발견**: "12*7" → `calculator(수식="12*8")` (숫자 틀림).
  과트리거 재확인(건강→search). → **v4 과제 확장**: ① 도구 예시 재균형 ② 직접답변 네거티브
  ③ **인자 정확도**(특히 숫자) 위해 calculator 등 다양한 인자 예시 대폭 증강.

## v2 — 사전학습 스케일업 (2026-06-24)
- **데이터 8~10배 확장**: 위키 50k → 400k 문서(`N_WIKI_DOCS`). 선택적 추가 코퍼스 셀(KoAlpaca,
  기본 OFF·opt-in) + 더 넣을 수 있는 템플릿. 데이터가 진짜 병목(57M 적정 ≈ 11억 토큰, 기존 ~1%).
- **학습량 확장 + LR 버그 수정**: LR 코사인 감쇠 지평을 `max_iters`(세션당) → **`total_iters`(전체, 40k)**
  로 분리. (기존엔 5000 step 이후 LR 이 min_lr 로 죽어 장기 학습 불가였음.)
- **체크포인트 latest/best 분리**: `otto_gpt.pt`(latest, 이어서 학습) + `otto_gpt_best.pt`(best val, 추론).
  매 eval 마다 latest 저장 → Colab 끊겨도 step 손실 없음.
- 토큰 예산·epoch 자동 추정 출력(과적합/과소학습 경고).
- 검증(로컬 CPU): 노트북 JSON 유효, 모델 forward/backward OK, LR 스케줄 수정 확인.
- **실측 결과(Colab, 2026-06-24)**: 40,000 step 완주, 56.7M params, **best val_loss 3.4233 (ppl ≈ 30.7)**.
  소형 모델 기준 양호. 생성 품질·추가학습 여부는 생성 샘플로 판단 예정.
  아키텍처(RoPE/RMSNorm/SwiGLU)는 이번엔 미변경.
