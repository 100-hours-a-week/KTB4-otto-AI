  → 폴백 → rag → **"월 구독료는 1만 9천 원"** 정답 교정. 정상 케이스(12*7/날씨/rag)는 otto 그대로.
- 정직한 범위: 57M 의 내부 환각 생성 자체는 못 막음. 다만 **사용자에게 도달하는 환각은 0** 으로 설계.

## v5 — 로컬 RAG 문장 단위 답변 (2026-06-24)
단락 통째 반환(v4) → **질문에 맞는 문장만** 골라 간결하게.
- 2단계 검색: ① 최적 단락(코퍼스 TF-IDF) → ② 단락 내 최적 문장.
- **문자 n-gram** 매칭으로 한국어 조사 흡수("구독료"~"구독료는"). 단어 토큰 idf 의
  조사 분절 문제 해결(이전엔 "구독료" idf 가 기본값으로 떨어져 답 문장을 놓침).
- 단락 **제목어(주제) down-weight(×0.3)** → 질문의 '의도'어가 이기게.
  ("어댑터즈 구독료 얼마야?" → 정의 문장 대신 "월 구독료는 1만 9천 원" 선택).
- **기본 1문장 압축**: 2번째 문장은 점수가 거의 동률(>=0.9)일 때만(예: 환불 정책처럼
  한 세트로 묶이는 답). 나머지는 핵심 1문장만.
- 검증(로컬, 7문항): 구독료/주차/수강생/수료/취업/업데이트 정확, 문서밖은 "찾을 수 없음".
  한계: 질문이 주제어를 반복하는 일부 케이스는 같은 단락의 관련 문장 반환(어휘 매칭 본질).
  진짜 의미 검색은 03_rag 임베딩(Gemini)이 담당.

## v4 — 오프라인 로컬 RAG 폴백 (2026-06-24)
Gemini quota/서버 없이도 in-domain 지식 질문에 답하도록 보강.
- `local_rag.py`: 03_rag 문서(docs/*.md)를 직접 읽어 **TF-IDF + 부분일치** 검색 (의존성 0, stdlib).
- `rag_search`: HTTP RAG 실패(서버/Gemini 없음) 시 → **로컬 문서 검색으로 자동 폴백**.
- `demo_offline.py`: 맥에서 즉시 실행 가능한 완전 오프라인 데모 (Gemini/Colab/서버 불필요).
- 검증(로컬): 어댑터즈 구독료/환불정책/수강생 → 근거 단락 정확 검색, 문서 밖("인공지능")은
  정직하게 "문서에서 찾을 수 없음". 에이전트 end-to-end 폴백 동작 + Colab 인라인 graceful.
- 효과: quota 대기 중에도 지식 질문 파이프라인(otto 라우팅 → rag → 근거답변)을 오프라인 검증 가능.

## v3 — 에이전트 인자 재추출 (2026-06-24)
otto_gpt 의 약점(정확한 인자 복사)을 **재학습 없이** 에이전트 레이어에서 보완.
- `extract.py`: 모델의 라우팅 결정은 신뢰, 정밀 인자는 **사용자 원문에서 직접 추출**.
  - calculator: 원문에서 산술식 정규식 추출 → 숫자 100% 정확
  - search: 꼬리말(검색해줘/알려줘) 제거
  - rag: 원문 질문 그대로(모델 인자 깨짐 회피)
  - weather: 모델 인자 유지(대체로 정확)
- `agent.run` 결과에 `model_args`(보정 전)/`args`(보정 후) 모두 포함(투명성).
- 검증(로컬, v4 실측 출력 기준 before→after): `346+679`→`345+678`=1023, `오늘 환율이`→`오늘 환율`,
  `인공학적`→`건강하게 사는 방법 알려줘`. 3개 실패 전부 보정.
- colab_agent_demo 에 extract 인라인 + 보정 표시.
